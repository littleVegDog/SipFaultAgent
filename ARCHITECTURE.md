# 系统架构与流程图

## 系统架构概述

本项目是一个基于RAG（检索增强生成）的智能问题定位助手，主要用于SBC（Session Border Controller）相关问题的定位和诊断。系统主要分为四个核心模块：主程序入口、查询路由系统、RAG检索系统和智能诊断Agent。

## 核心组件

### 1. 主程序入口 (main.py)
- 系统入口点和控制中心
- 初始化所有核心组件：LLM客户端、RAG系统、工具集、输入增强器等
- 实现用户交互循环，处理输入并分发到相应处理流程

### 2. 查询路由系统 (router.py)
- 识别用户查询类型（协议查询 vs 故障诊断）
- 基于关键词匹配来判断查询场景
- 协议查询关键词：是什么、含义、解释、说明、定义等
- 故障诊断关键词：失败、错误、问题、故障、怎么、如何等

### 3. RAG检索系统 (rag.py)
- 核心检索增强生成模块
- 使用 ChromaDB 作为向量存储后端
- 支持语义搜索（ChromaDB 向量检索）
- 提供文档过滤和重新排序功能
- 支持 Parent-Child 文档块结构（章节关系）
- 实现增量文档添加功能
- 分批构建知识库，支持断点续建

### 4. 智能诊断Agent (agent.py)
- 基于LLM的故障诊断智能体
- 实现记忆管理（上下文对话）
- 支持工具调用机制
- 包含JSON响应解析和错误恢复机制

## 数据流程

```
用户输入查询
    ↓
[Query Router] → 判断查询类型
    ↓
协议查询 → [RAG System] → [LLM] → 结果输出
故障诊断 → [Agent] → [RAG System] → [Tools] → [LLM] → 结果输出
```
